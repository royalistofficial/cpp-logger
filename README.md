# Многопоточная запись сообщений в журнал (C++17)

Динамическая библиотека логирования и консольное приложение к ней.
Приложение читает сообщение с необязательным уровнем важности, передаёт его
в отдельный поток записи и сразу возвращается к вводу.

Только STL, стандарт C++17, сборка `g++` через `make`, целевая ОС —
Ubuntu/Debian.

## Сборка

```sh
make            # библиотека и приложение
make lib        # только liblogger.so
make app        # только приложение
make test       # собрать и прогнать тесты (85 шт.)
make clean      # удалить продукты сборки

./bin/logger_app app.log INFO
```

Библиотека подключается динамически, путь зашит через `$ORIGIN` —
`LD_LIBRARY_PATH` не нужен.

## Ввод

Параметры запуска: файл журнала и, необязательно, уровень по умолчанию
(`INFO`, `WARNING`, `ERROR`; регистр не важен, есть краткие `WARN`/`ERR`).

| Ввод | Результат |
|---|---|
| `диск заполнен` | сообщение с текущим уровнем по умолчанию |
| `error диск заполнен` | сообщение с уровнем ERROR |
| `\error диск заполнен` | текст «error диск заполнен» дословно |
| `:level warning` | сменить уровень по умолчанию |
| `:help`, `:quit` | справка, выход (то же делает Ctrl+D) |

Формат записи — одна запись в одной строке, время местное:

```
[2026-07-18 14:02:36.313] [WARNING] диск заполнен
```

## Структура

```
include/logger/   публичный API библиотеки
src/logger/       приватная реализация
app/              консольное приложение
tests/            тесты и минимальный фреймворк
```

### Библиотека

| Файл | Роль |
|---|---|
| `ILogger.hpp` | интерфейс журнала: `log()`, смена уровня |
| `ILogSink.hpp` | куда пишем (файл, память, сокет) |
| `ILogFormatter.hpp` | как выглядит запись |
| `LogLevel.hpp`, `LogRecord.hpp` | уровни важности и данные записи |
| `Logger.hpp/.cpp` | политика: порог и синхронизация |
| `FileLogSink` *(скрыт)* | запись в файл |
| `DefaultFormatter` *(скрыт)* | формат по умолчанию |

`Logger` отвечает только за порог и сериализацию доступа, а представление и
ввод-вывод делегирует интерфейсам — формат или назначение меняются без правок
в нём. Из `.so` экспортируется ровно публичный API
(`-fvisibility=hidden` + макрос `LOGGER_API`), реализации наружу не видны.

Уровень хранится в `std::atomic`, форматирование идёт вне мьютекса — под
блокировкой остаётся только обращение к приёмнику.

### Приложение

```
stdin → Application → IMessageSink → BlockingQueue → поток записи → ILogger → файл
        разбор ввода                 ограниченная    AsyncLogWriter
        и команд                     ёмкость
```

| Файл | Роль |
|---|---|
| `main.cpp` | связывает компоненты, коды возврата |
| `CliOptions` | аргументы командной строки |
| `Application` | цикл чтения консоли |
| `CommandProcessor` | команды `:quit`, `:level`, `:help` |
| `InputParser` | разбор строки на уровень и текст |
| `IMessageSink` | абстракция доставки сообщений |
| `AsyncLogWriter` | очередь и поток записи за этой абстракцией |
| `BlockingQueue` | потокобезопасная очередь с ограниченной ёмкостью |

Ключевые решения:

* `Application` зависит только от `IMessageSink`, ничего не создаёт и не
  владеет потоками — связывание вынесено в `main`. Цикл ввода тестируется без
  файлов и потоков.
* Ёмкость очереди ограничена: при медленном диске поток ввода ждёт, а память
  не растёт без предела.
* Сообщения и смена уровня едут одной очередью (`std::variant`), поэтому
  порядок «сменил уровень — пишу по новому порогу» соблюдается без
  дополнительной синхронизации.
* Время получения снимается до обработки и передаётся в журнал явно — в файл
  попадает момент ввода, а не момент записи.
* Команды заданы одной таблицей, из неё же строится справка.
